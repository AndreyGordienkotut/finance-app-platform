Цей проєкт — бекенд мікросервісна система для керування фінансами користувача, побудована з орієнтацією на продакшн-підхід.
Система дозволяє користувачам керувати рахунками, виконувати фінансові операції, працювати з лімітами, категоріями витрат, отримувати аналітику та статистику, а також виконувати конвертацію валют.
Проєкт реалізовано як набір незалежних сервісів, що взаємодіють між собою через REST API, та готовий до інтеграції з будь-яким frontend-додатком або мобільним клієнтом.

Архітектура

Проєкт побудований за принципами мікросервісної архітектури:

- eureka-service — service discovery
- auth-service — автентифікація та авторизація
- account-service — керування рахунками користувача
- transaction-service — фінансові транзакції
  -  analytics-service — статистика та аналітика витрат
  - exchange-rate-service — отримання та кешування валютних курсів

Комунікація між сервісами реалізована через REST + OpenFeign.

Основний функціонал 

Auth Service

- Реєстрація користувачів
- Авторизація та аутентифікація
- JWT-based security:
  - Access Token
  - Refresh Token
- Оновлення access-токенів без повторного логіну
- Logout з інвалідацією токенів

Account Service
- Створення рахунків користувача
- Підтримка кількох валют
- Зміна статусу рахунку (ACTIVE / CLOSED)
- Перевірка прав доступу до рахунків
- Інтеграція з transaction-service

Transaction Service
- Проведення фінансових операцій:
  -TRANSFER
  - DEPOSIT
  - WITHDRAW
- Idempotency-підхід для запобігання дублюванню транзакцій
- Перевірка лімітів користувача
- Збереження історії транзакцій
- Підтримка категорій витрат
- Робота з декількома валютами
- Інтеграція з exchange-rate-service
- Cache eviction аналітичних даних після мутацій

Analytics Service
Аналітичний сервіс для побудови фінансової статистики що є частиною Transaction Service
- Загальна сума витрат за період
- Топ категорій витрат
- Таймлайн витрат (day / week / month)
- Фільтрація по датах
- Оптимізовані SQL-запити
- Кешування важких запитів через Redis
- TTL для аналітичних даних

Exchange Rate Service
Отримання актуальних валютних курсів з зовнішнього API що э частиною Transaction Service
- Обробка помилок зовнішнього сервісу
- Кешування валютних курсів у Redis
- TTL для курсів валют
- Мінімізація кількості зовнішніх запитів

Архітектурні рішення

- Трирівнева архітектура: Controller → Service → Repository
- DTO для API контрактів
- Чітке розділення відповідальностей
- SOLID-підхід
- Exception Handling через @ControllerAdvice
- Валідація вхідних даних
- Логування ключових бізнес-операцій

Додаткова інформація
- Проєкт орієнтований на продакшн-підхід
- Сервіси незалежні та масштабовані
- Кешування використовується для зменшення навантаження на БД
- API документоване через Swagger
- Архітектура дозволяє легко додавати нові сервіси та функціонал
- Архітектура проєкту підготовлена до контейнеризації (Redis, БД, конфігурації), однак на поточний момент застосунок запускається локально для спрощення розробки та дебагу.
